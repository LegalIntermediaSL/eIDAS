# 43. Criterios terminologicos del proyecto

## Introduccion

Este documento fija algunos criterios de lenguaje y enfoque para mantener coherencia en el tutorial.

## 1. Uso de "eIDAS"

Se utiliza `eIDAS` para referirse:

- al Reglamento (UE) 910/2014 en sentido estricto
- al marco europeo de identificacion electronica y servicios de confianza en sentido amplio, cuando el contexto lo permite

## 2. Uso de "eIDAS 2.0"

Se utiliza `eIDAS 2.0` como expresion divulgativa para la reforma introducida por el Reglamento (UE) 2024/1183.

## 3. Uso de "firma simple"

Se admite la expresion `firma electronica simple` como formula didactica, pero se recuerda que la categoria formal de partida en eIDAS es la `firma electronica`.

## 4. Uso de "transposicion" en Espana

El proyecto evita hablar de "transposicion" estricta del Reglamento eIDAS, porque un reglamento europeo es directamente aplicable. Se prefiere hablar de:

- desarrollo
- complemento
- adaptacion del marco nacional

## 5. Uso de "prestador cualificado"

La palabra `cualificado` se usa como categoria juridica concreta y no como elogio o adjetivo comercial.

## 6. Uso de "wallet europea"

Se utiliza como forma corta de `Cartera Europea de Identidad Digital` o `EUDI Wallet`, siempre entendida como parte del nuevo marco europeo de identidad digital.

## 7. Uso de "lista de confianza" y "TSL"

Se admite el uso alterno de `lista de confianza` y `TSL`, pero se procura que la primera mension completa sea `lista de confianza (TSL)` para facilitar la lectura a perfiles no tecnicos.

## 8. Criterio general del tutorial

El proyecto adopta un lenguaje:

- didactico
- riguroso en lo esencial
- simplificado cuando la complejidad tecnica o juridica lo aconseja

## Resumen rapido

Mantener criterios terminologicos estables ayuda a que el tutorial sea mas claro, menos ambiguo y mas facil de ampliar con el tiempo.
