# 30. Errores frecuentes de implantacion

## Introduccion

Implantar herramientas vinculadas a eIDAS no consiste solo en comprar tecnologia. Muchos errores aparecen por simplificaciones excesivas o por falta de coordinacion entre lo juridico, lo tecnico y lo operativo.

## Error 1. Pensar solo en la firma

Muchas organizaciones reducen eIDAS a la firma electronica y olvidan:

- sellos
- sellos de tiempo
- entrega electronica certificada
- validacion y conservacion

## Error 2. Elegir la herramienta sin analizar el proceso

No debe empezarse por la herramienta, sino por:

- el proceso
- el riesgo
- la necesidad de prueba

## Error 3. Usar servicios mas debiles de lo necesario

En procesos sensibles, una herramienta insuficiente puede generar problemas futuros de prueba o impugnacion.

## Error 4. Sobredimensionar todo

El error contrario tambien existe: aplicar siempre la solucion mas formal y costosa, incluso cuando no aporta valor proporcional.

## Error 5. No verificar al prestador

Es fundamental comprobar:

- si el prestador es cualificado o no
- que servicio concreto ofrece
- si el servicio figura oficialmente como cualificado

## Error 6. No pensar en el largo plazo

Firmar hoy no basta si manana no puede validarse la evidencia o no se ha conservado correctamente.

## Error 7. Falta de coordinacion interna

Si el area juridica, la tecnica y la operativa trabajan por separado, es facil implantar soluciones que no encajan con el proceso real.

## Error 8. Confundir marketing con categoria juridica

Palabras como `seguro`, `digital`, `certificado` o `avanzado` no significan automaticamente que exista un servicio cualificado en sentido eIDAS.

## Error 9. No formar a los usuarios

Incluso una buena implantacion puede fallar si quienes usan el sistema no entienden:

- cuando usar cada herramienta
- que evidencia se genera
- que riesgos existen si se usa mal

## Error 10. No revisar el sistema periodicamente

La implantacion no deberia verse como un acto unico, sino como un sistema que debe revisarse conforme cambian:

- procesos
- riesgo
- tecnologia
- marco regulatorio

## Resumen rapido

Los errores mas comunes aparecen cuando se ignora el riesgo, no se verifica al prestador, se descuida la prueba futura o se implanta tecnologia sin verdadero encaje en el proceso.
